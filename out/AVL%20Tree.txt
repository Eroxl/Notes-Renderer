<p>AVL tree's are a type of self-<a href="./Balanced Binary Tree">balancing binary trees</a> where each node's left and right subtrees differ in height by at most only 1. AVL trees perform their auto balancing by performing &quot;rotations&quot; on insertion and deletion depending on where affected nodes were/are located.</p>
<p>To support quick inserts and removals, AVL tree's typically store the height of each node alongside the node's data. This height is then recalculated and updated for a few specific nodes after rotations.</p>
<h1>Imbalance</h1>
<p>After a node is inserted or deleted from the <a href="./Tree (Data Structure)">tree</a> it's predecessor can become &quot;imbalanced&quot; (the difference between the heights of it's left and right children become greater than 1).</p>
<p>With the assumption that the <a href="./Tree (Data Structure)">tree</a> was balanced before an operation there can only be 4 different types of imbalance.</p>
<h2>Left-Left Imbalance</h2>
<p>A left-left imbalance occurs when a node’s left subtree is taller than its right subtree by more than 1, and the extra height comes from the left child’s left subtree. A left-left imbalance can be fixed by performing a single right rotation on the root node.</p>

<h1>Rotations</h1>
<p>Rotations are operations that can be performed on AVL tree's which change the shape of the <a href="./Tree (Data Structure)">tree</a> while maintaining the properties of a <a href="./Binary Search Tree">binary search tree</a>.</p>
<h2>Left Rotation</h2>
<p>Given a subtrees root with a right sub child a left rotation can be performed in the following steps:</p>
<ol>
<li>Create a temporary reference to the old root's right child as <code>temp</code>.</li>
<li>Update the old root's right child to be <code>temp</code>'s left child (if it exists).</li>
<li>Update <code>temp</code>'s left child to be the old root.</li>
<li>If the old root had a parent update it's references to <code>temp</code> accordingly.</li>
</ol>
<h2>Right Rotation</h2>
<p>Given a subtrees root with a left sub child a right rotation can be performed in the following steps:</p>
<ol>
<li>Create a temporary reference to the old root's left child as <code>temp</code>.</li>
<li>Update the old root's left child to be <code>temp</code>'s right child (if it exists).</li>
<li>Update <code>temp</code>'s right child to be the old root.</li>
<li>If the old root had a parent update it's references to <code>temp</code> accordingly.</li>
</ol>
